## ADDED Requirements

### Requirement: Регистрация картины рейтинга
Система MUST предоставлять отдельный Trade Rush предмет картины рейтинга команд с идентификатором `trade-rush:team_rating_painting`.

#### Scenario: Картина доступна как предмет мода
- **WHEN** мод инициализируется
- **THEN** предмет картины рейтинга зарегистрирован в пространстве имён Trade Rush с ID `trade-rush:team_rating_painting`

#### Scenario: Картина доступна для creative/admin workflow
- **WHEN** игрок или администратор использует creative/admin workflow мода
- **THEN** картину рейтинга можно получить как предмет мода для тестирования или выдачи игрокам

#### Scenario: Картина не является блоком управления командами
- **WHEN** картина рейтинга зарегистрирована как игровой объект
- **THEN** она доступна как item, который можно держать в руке и размещать на стене
- **AND** она не предоставляет UI управления командами сама по себе

### Requirement: Картина размещается на стене
Система MUST позволять игроку размещать картину рейтинга как настенный world-space объект размером 4×4 блока.

#### Scenario: Игрок размещает картину на подходящей стене
- **WHEN** server-side игрок использует предмет `trade-rush:team_rating_painting` по вертикальной поверхности
- **AND** за поверхностью есть поддержка и свободная область для картины размером 4×4 блока
- **THEN** система создаёт размещённую картину рейтинга в мире
- **AND** размещённая картина ориентирована по выбранной стене

#### Scenario: Картина не размещается без места или поддержки
- **WHEN** игрок использует предмет `trade-rush:team_rating_painting` по поверхности
- **AND** область 4×4 блока не поддержана стеной или пересекается с препятствиями
- **THEN** система не создаёт размещённую картину рейтинга
- **AND** состояние команд не изменяется

#### Scenario: Предмет расходуется только при успешном размещении
- **WHEN** игрок в survival-like режиме успешно размещает картину рейтинга
- **THEN** количество предметов `trade-rush:team_rating_painting` у игрока уменьшается на один

#### Scenario: Creative placement не расходует предмет
- **WHEN** игрок в creative-like режиме успешно размещает картину рейтинга
- **THEN** предмет `trade-rush:team_rating_painting` не расходуется

#### Scenario: Разрушение картины возвращает предмет
- **WHEN** размещённая картина рейтинга разрушена обычным игровым способом
- **THEN** размещённая картина удаляется из мира
- **AND** система дропает предмет `trade-rush:team_rating_painting`, если правила режима игры позволяют дроп

#### Scenario: Потеря поддержки удаляет картину
- **WHEN** размещённая картина рейтинга теряет стену или поддержку, необходимую для существования
- **THEN** размещённая картина удаляется из мира
- **AND** система обрабатывает дроп так же, как при обычном разрушении картины

### Requirement: Картина отображает рейтинг команд
Размещённая картина рейтинга MUST отображать read-only рейтинг команд по очкам от большего количества очков к меньшему.

#### Scenario: Строка рейтинга содержит место, команду и очки
- **WHEN** размещённая картина рейтинга отображает команду
- **THEN** строка этой команды содержит номер места
- **AND** строка содержит название команды
- **AND** строка содержит количество очков команды

#### Scenario: Команды отсортированы по рейтингу
- **WHEN** размещённая картина рейтинга отображает несколько команд с разным количеством очков
- **THEN** команда с большим количеством очков отображается выше команды с меньшим количеством очков

#### Scenario: Рейтинг соответствует порядку TeamService
- **WHEN** сервер строит snapshot для картины рейтинга
- **THEN** порядок команд основан на результате `TeamService.listTeams()`
- **AND** renderer картины не применяет отдельную client-side сортировку команд

#### Scenario: Обычная нумерация при равных очках
- **WHEN** две или больше команд имеют одинаковое количество очков
- **THEN** картина рейтинга всё равно присваивает им последовательные места по порядку отображения, например `1`, `2`, `3`
- **AND** картина не объединяет команды с одинаковыми очками в одно общее место

#### Scenario: Картина открывается без команд
- **WHEN** в мире размещена картина рейтинга
- **AND** команды ещё не созданы
- **THEN** картина отображает empty state о том, что команд пока нет
- **AND** клиент не падает из-за пустого рейтинга

### Requirement: Картина показывает верхнюю часть рейтинга
Размещённая картина рейтинга MUST показывать только первые команды, которые помещаются в layout картины, и MUST NOT предоставлять страницы, scroll или другие элементы навигации.

#### Scenario: Команд меньше видимого лимита
- **WHEN** количество команд меньше или равно видимому лимиту картины рейтинга
- **THEN** картина отображает все команды из snapshot рейтинга

#### Scenario: Команд больше видимого лимита
- **WHEN** количество команд больше видимого лимита картины рейтинга
- **THEN** картина отображает только первые команды рейтинга до видимого лимита
- **AND** команды ниже видимого лимита не отображаются на картине

#### Scenario: Видимый лимит стабилен для всех клиентов
- **WHEN** разные клиенты видят одну и ту же размещённую картину рейтинга
- **THEN** они используют одинаковый фиксированный лимит видимых строк рейтинга
- **AND** состав отображаемого top-рейтинга не зависит от GUI scale, расстояния до картины или размера окна клиента

#### Scenario: На картине нет перелистывания
- **WHEN** рейтинг содержит больше команд, чем помещается на картине
- **THEN** картина не отображает кнопки следующей или предыдущей страницы
- **AND** картина не предоставляет scroll или другие элементы управления списком

### Requirement: Картина автоматически обновляется при изменении команд
Размещённая картина рейтинга MUST автоматически отражать server-side изменения командного состояния, влияющие на список команд, названия команд или очки.

#### Scenario: Очки меняются после размещения картины
- **WHEN** в мире есть размещённая картина рейтинга
- **AND** после этого очки одной или нескольких команд изменились на сервере
- **THEN** сервер строит новый snapshot рейтинга для картины
- **AND** online clients получают обновление рейтинга без действий игрока
- **AND** видимая картина обновляет отображаемые места, названия команд и очки

#### Scenario: Команда создаётся после размещения картины
- **WHEN** в мире есть размещённая картина рейтинга
- **AND** после этого на сервере создаётся команда
- **THEN** картина рейтинга отражает актуальный список команд после server push обновления

#### Scenario: Команда переименовывается после размещения картины
- **WHEN** в мире есть размещённая картина рейтинга
- **AND** после этого на сервере переименовывается команда
- **THEN** картина рейтинга отображает новое название команды после server push обновления

#### Scenario: Команда удаляется после размещения картины
- **WHEN** в мире есть размещённая картина рейтинга
- **AND** после этого на сервере удаляется команда
- **THEN** картина рейтинга больше не отображает удалённую команду после server push обновления

#### Scenario: Игрок подключается после изменения рейтинга
- **WHEN** игрок подключается к серверу после изменения командного рейтинга
- **THEN** сервер отправляет этому игроку актуальный snapshot картины рейтинга
- **AND** размещённые картины, которые игрок видит после подключения, отображают актуальный рейтинг

### Requirement: Обновления картины масштабируются независимо от количества картин
Система MUST синхронизировать рейтинг картины через общий top-rating snapshot для клиентов и MUST NOT умножать сетевые обновления на количество размещённых картин рейтинга.

#### Scenario: Несколько картин используют один snapshot клиента
- **WHEN** клиент получил snapshot картины рейтинга
- **AND** в загруженных чанках клиента есть несколько размещённых картин рейтинга
- **THEN** все эти картины используют полученный snapshot для отображения рейтинга
- **AND** каждая картина не требует отдельного server payload с теми же строками рейтинга

#### Scenario: Изменение рейтинга не отправляет payload на каждую картину
- **WHEN** server-side состояние команд изменяется
- **AND** в мире размещено несколько картин рейтинга
- **THEN** сервер строит общий top-rating snapshot для картины рейтинга
- **AND** сервер отправляет не более одного такого snapshot на подходящего online client для этого изменения рейтинга

#### Scenario: Несколько картин показывают согласованный рейтинг
- **WHEN** в мире размещено несколько картин рейтинга
- **AND** клиент видит две или больше таких картин одновременно
- **THEN** эти картины отображают одинаковый актуальный top-рейтинг

### Requirement: Картина рейтинга не управляет командами
Размещённая картина рейтинга MUST быть read-only world-space display и MUST NOT предоставлять действия управления командами.

#### Scenario: Взаимодействие с картиной не меняет команды
- **WHEN** игрок взаимодействует с размещённой картиной рейтинга
- **THEN** состояние команд, участников и очков не изменяется из-за самого взаимодействия

#### Scenario: На картине нет действий управления командами
- **WHEN** игрок смотрит на размещённую картину рейтинга
- **THEN** картина не предоставляет действия создания команды
- **AND** картина не предоставляет действия присоединения к команде или выхода из команды
- **AND** картина не предоставляет действия переименования или удаления команды
- **AND** картина не предоставляет действия изменения очков

#### Scenario: Автообновление рейтинга не меняет команды
- **WHEN** сервер отправляет обновлённый snapshot картины рейтинга клиентам
- **THEN** состояние команд, участников и очков не изменяется из-за самого UI/world-space обновления

### Requirement: Документация картины рейтинга
Документация проекта MUST кратко описывать предмет картины рейтинга и его базовое поведение.

#### Scenario: README содержит описание картины рейтинга
- **WHEN** пользователь читает `README.md`
- **THEN** README содержит раздел или пункт о `Team Rating Painting`
- **AND** README указывает ID предмета `trade-rush:team_rating_painting`
- **AND** README описывает, что картина размещается на стене и отображает верхнюю часть рейтинга команд по очкам
- **AND** README описывает, что картина автоматически обновляет рейтинг при server-side изменении командного состояния
