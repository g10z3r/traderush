## Why

Игрокам нужен атмосферный способ посмотреть рейтинг команд без чат-команд и без перегруженного UI управления командами. Vision проекта описывает отдельную книгу рейтинга как системный предмет, который игрок берёт в руку и открывает для просмотра списка команд и их очков.

## What Changes

- Добавить отдельный Trade Rush предмет `TeamRatingBook` с идентификатором `trade-rush:team_rating_book`.
- Сделать книгу нестакаемой и доступной через creative/admin workflow для тестирования и выдачи.
- При использовании книги в руке открывать read-only Minecraft UI рейтинга команд.
- При открытии книги на сервере получать актуальный snapshot команд через существующий `TeamService.listTeams()`.
- Показывать в UI место команды, название команды и количество очков.
- Использовать обычную нумерацию мест: `1`, `2`, `3` и далее, даже если у нескольких команд одинаковые очки.
- Если команды не помещаются на одной странице книги, отображать рейтинг постранично с book-like перелистыванием страниц.
- Автоматически отправлять новый snapshot в открытую книгу рейтинга при server-side изменении состояния команд, включая изменение очков.
- Не добавлять в книгу action-кнопки, поля ввода или функции создания, присоединения, выхода, переименования или удаления команд; элементы перелистывания страниц допустимы только как навигация по книге.
- Не реализовывать в рамках этой change бесконечный сундук на спавне для получения книги.
- Кратко описать книгу рейтинга в `README.md` рядом с существующим описанием системных объектов мода.

## Capabilities

### New Capabilities

- `team-rating-book`: системный предмет книги рейтинга, открывающий read-only UI с рейтингом команд по очкам и автоматически обновляющий открытый UI при изменении командного состояния.

### Modified Capabilities

Нет.

## Impact

- Новая регистрация предмета в пространстве имён Trade Rush и добавление его в creative/admin workflow.
- Новый server-side путь использования предмета и открытия menu/screen.
- Новый client-side экран рейтинга команд с постраничным book-like отображением длинного списка.
- Новый snapshot и clientbound networking для передачи рейтинга от сервера клиенту при открытии книги и при server-side изменениях команд.
- Переиспользование существующего `TeamService.listTeams()` как источника порядка команд и очков.
- Новые localization strings и минимальные item assets/model для книги.
- Обновление `README.md`.
