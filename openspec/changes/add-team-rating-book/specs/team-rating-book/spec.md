## ADDED Requirements

### Requirement: Регистрация книги рейтинга
Система MUST предоставлять отдельный Trade Rush предмет книги рейтинга команд с идентификатором `trade-rush:team_rating_book`.

#### Scenario: Книга доступна как предмет мода
- **WHEN** мод инициализируется
- **THEN** предмет книги рейтинга зарегистрирован в пространстве имён Trade Rush с ID `trade-rush:team_rating_book`

#### Scenario: Книга не является блоком
- **WHEN** книга рейтинга зарегистрирована как игровой объект
- **THEN** она доступна как item, который можно держать в руке
- **AND** она не регистрируется как block

#### Scenario: Книга не стакается
- **WHEN** игрок получает несколько книг рейтинга
- **THEN** каждая книга занимает отдельный stack с максимальным размером 1

#### Scenario: Книга доступна для creative/admin workflow
- **WHEN** игрок или администратор использует creative/admin workflow мода
- **THEN** книгу рейтинга можно получить как предмет мода для тестирования или выдачи игрокам

### Requirement: Использование книги открывает рейтинг команд
Система MUST открывать read-only UI рейтинга команд, когда игрок использует книгу рейтинга в руке.

#### Scenario: Игрок открывает книгу из руки
- **WHEN** server-side игрок использует предмет `trade-rush:team_rating_book`, находящийся в руке
- **THEN** система открывает для этого игрока UI книги рейтинга

#### Scenario: Рейтинг строится при открытии книги
- **WHEN** UI книги рейтинга открывается
- **THEN** сервер получает актуальный список команд через существующий доменный путь команд
- **AND** snapshot рейтинга отражает состояние команд на момент открытия книги

#### Scenario: Книга открывается без команд
- **WHEN** игрок открывает книгу рейтинга, а команды ещё не созданы
- **THEN** UI книги рейтинга открывается без ошибки
- **AND** UI показывает empty state о том, что команд пока нет

### Requirement: Отображение рейтинга команд
UI книги рейтинга MUST отображать команды как рейтинг по очкам от большего количества очков к меньшему.

#### Scenario: Строка рейтинга содержит место, команду и очки
- **WHEN** UI книги рейтинга отображает команду
- **THEN** строка этой команды содержит номер места
- **AND** строка содержит название команды
- **AND** строка содержит количество очков команды

#### Scenario: Команды отсортированы по очкам
- **WHEN** UI книги рейтинга отображает несколько команд с разным количеством очков
- **THEN** команда с большим количеством очков отображается выше команды с меньшим количеством очков

#### Scenario: Обычная нумерация при равных очках
- **WHEN** две или больше команд имеют одинаковое количество очков
- **THEN** UI всё равно присваивает им последовательные места по порядку отображения, например `1`, `2`, `3`
- **AND** UI не объединяет команды с одинаковыми очками в одно общее место

#### Scenario: Рейтинг соответствует порядку TeamService
- **WHEN** сервер строит snapshot для книги рейтинга
- **THEN** порядок команд основан на результате `TeamService.listTeams()`
- **AND** UI не применяет отдельную client-side сортировку команд

### Requirement: Длинный рейтинг отображается страницами
UI книги рейтинга MUST предоставлять book-like постраничное отображение, когда количество команд превышает вместимость одной страницы.

#### Scenario: Команды не помещаются на одной странице
- **WHEN** UI книги рейтинга получает snapshot с количеством команд больше вместимости одной страницы
- **THEN** UI разделяет рейтинг на несколько страниц
- **AND** игрок может перейти к следующей и предыдущей странице рейтинга
- **AND** каждая команда из snapshot доступна на одной из страниц рейтинга

#### Scenario: Навигация страниц не является действием управления командами
- **WHEN** UI книги рейтинга предоставляет элементы перелистывания страниц
- **THEN** эти элементы только меняют видимую страницу рейтинга
- **AND** они не запрашивают изменение команд, участников или очков

#### Scenario: Snapshot изменил количество страниц
- **WHEN** открытая книга рейтинга получает обновлённый snapshot от сервера
- **AND** текущий номер страницы больше нового количества страниц
- **THEN** UI безопасно переходит на последнюю доступную страницу или empty state
- **AND** UI не показывает пустую несуществующую страницу

### Requirement: Snapshot книги обновляется при открытии и изменении команд
Открытый UI книги рейтинга MUST показывать snapshot, полученный при открытии книги, и MUST автоматически получать новый snapshot от сервера при изменении server-side состояния команд, влияющего на список команд, их названия или очки.

#### Scenario: Очки меняются после открытия книги
- **WHEN** игрок открыл книгу рейтинга
- **AND** после этого очки одной или нескольких команд изменились на сервере
- **THEN** сервер строит новый snapshot рейтинга
- **AND** сервер отправляет обновлённый snapshot игроку с открытой книгой рейтинга
- **AND** UI книги рейтинга обновляет отображаемые места, названия команд и очки без действий игрока

#### Scenario: Команда создаётся, переименовывается или удаляется после открытия книги
- **WHEN** игрок открыл книгу рейтинга
- **AND** после этого на сервере создаётся, переименовывается или удаляется команда
- **THEN** сервер строит новый snapshot рейтинга
- **AND** сервер отправляет обновлённый snapshot игроку с открытой книгой рейтинга
- **AND** UI книги рейтинга отражает актуальный список команд без действий игрока

#### Scenario: Повторное открытие также показывает свежий рейтинг
- **WHEN** игрок закрывает книгу рейтинга после изменения состояния команд
- **AND** затем открывает книгу рейтинга снова
- **THEN** сервер строит новый snapshot
- **AND** UI показывает рейтинг на основе актуального состояния команд в момент повторного открытия

### Requirement: Книга рейтинга не управляет командами
UI книги рейтинга MUST быть read-only и MUST NOT предоставлять действия управления командами.

#### Scenario: В книге нет действий управления командами
- **WHEN** игрок открыл UI книги рейтинга
- **THEN** UI не предоставляет действия создания команды
- **AND** UI не предоставляет действия присоединения к команде или выхода из команды
- **AND** UI не предоставляет действия переименования или удаления команды

#### Scenario: В книге нет action-кнопок и полей ввода
- **WHEN** игрок открыл UI книги рейтинга
- **THEN** UI не предоставляет action-кнопки вроде `Refresh`, создания команды или управления командой
- **AND** UI не предоставляет поля ввода
- **AND** элементы перелистывания страниц допустимы, если рейтинг занимает больше одной страницы

#### Scenario: Просмотр рейтинга не меняет состояние команд
- **WHEN** игрок открывает и просматривает книгу рейтинга
- **THEN** состояние команд, участников и очков не изменяется из-за самого просмотра книги

#### Scenario: Автообновление рейтинга не меняет состояние команд
- **WHEN** сервер отправляет обновлённый snapshot в открытую книгу рейтинга
- **THEN** состояние команд, участников и очков не изменяется из-за самого UI-обновления

### Requirement: Документация книги рейтинга
Документация проекта MUST кратко описывать предмет книги рейтинга и его базовое поведение.

#### Scenario: README содержит описание книги рейтинга
- **WHEN** пользователь читает `README.md`
- **THEN** README содержит раздел или пункт о `Team Rating Book`
- **AND** README указывает ID предмета `trade-rush:team_rating_book`
- **AND** README описывает, что книга открывает read-only рейтинг команд по очкам
- **AND** README описывает, что открытая книга автоматически обновляет рейтинг при server-side изменении командного состояния
- **AND** README описывает, что длинный рейтинг отображается страницами
