## Почему

Управление командами сейчас доступно через чат-команды, но по vision проекта основные игровые действия должны происходить через объект в мире, чтобы не ломать атмосферу и UX Minecraft. Нужен специальный блок на спавне, через который игроки смогут смотреть команды и выполнять базовые операции управления без ввода команд в чат.

## Что меняется

- Добавить специальный блок управления командами, размещаемый в мире.
- При взаимодействии игрока с блоком открывать Minecraft UI для управления командами.
- В UI показывать список команд, очки, количество участников и участников выбранной команды.
- Позволить игроку через UI:
  - создать команду;
  - вступить в выбранную команду;
  - выйти из текущей команды;
  - удалить пустую команду;
  - переименовать пустую команду как первую форму редактирования команды.
- Добавить чат-команду переименования команды, чтобы доменная операция `renameTeam` была доступна и через `/traderush team`/`/tr team`, а не только через UI.
- Использовать существующий доменный слой `TeamService` для всех операций, чтобы чат-команды и UI разделяли одни правила и сохранение состояния.
- Показывать игроку результат операции или понятную ошибку.
- Не добавлять `force delete` в обычный UI; принудительное удаление остаётся административным/отладочным сценарием вне этой фичи.

## Возможности

### Новые возможности

- `team-management-ui`: управление командами через специальный блок в мире и связанный Minecraft UI.

### Изменяемые возможности

Нет.

## Влияние

- Новая регистрация блока и предмета блока в `main` source set.
- Новый серверный слой UI/menu для открытия интерфейса от блока и получения актуального снимка состояния команд.
- Новый client-side экран для отображения команд и отправки действий игрока.
- Новые client/server network payloads для операций UI и обновления состояния.
- Расширение доменного слоя команд, если для редактирования потребуется `renameTeam`, и подключение этой операции к chat-command layer.
- Новые assets для блока/предмета и, возможно, локализационные строки.
- Существующие чат-команды должны продолжить работать как резервный, административный и отладочный путь.
