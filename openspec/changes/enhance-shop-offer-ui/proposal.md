## Why

Текущий UI магазина показывает список заказов и требуемые предметы, но временные заказы не дают игроку понять, сколько времени осталось до истечения. Кроме того, миниатюры требуемых блоков/предметов слишком маленькие и не всегда однозначны без стандартного tooltip.

## What Changes

- Добавить отображение countdown-таймера для временных заказов в левой колонке списка заказов.
- Добавить синхронизированное отображение того же countdown-таймера в правой панели выбранного временного заказа.
- Не показывать таймеры для limited-заказов.
- Показывать стандартный vanilla item tooltip при hover по миниатюрам требуемых предметов/блоков в списке и в детальной панели.
- Защитить trade-flow от списания предметов, если временный заказ истёк, пока UI был открыт.
- Обновить сетевой snapshot магазина так, чтобы клиент получал достаточно данных для корректного локального countdown.

## Capabilities

### New Capabilities
- `shop-offer-ui`: Поведение клиентского UI магазина для отображения заказов, таймеров временных заказов и tooltip требуемых предметов.

### Modified Capabilities

Нет существующих capabilities для изменения.

## Impact

- Клиентский экран: `traderush/src/client/java/traderush/client/ui/shop/ShopOffersScreen.java`.
- Сетевые DTO/payload: `ShopOfferEntry`, `ShopOffersPayload`.
- Серверная сборка snapshot: `ShopNetworking.sendOffers(...)` / `buildOfferEntries(...)`.
- Доменная модель временного активного заказа: `TimedActiveOffer` может потребовать getter для `endsAtTick`.
- Trade handler: `ShopNetworking.handleTrade(...)` должен проверять активность заказа до удаления предметов.
- Зависимости не меняются.
