## ADDED Requirements

### Requirement: Snapshot содержит время истечения временных заказов
Система SHALL отправлять клиенту snapshot магазина с серверным тиком snapshot-а и временем истечения для каждого временного заказа.

#### Scenario: Временный заказ содержит данные истечения
- **WHEN** сервер формирует `ShopOffersPayload` для магазина с активным timed-заказом
- **THEN** соответствующий `ShopOfferEntry` содержит tick истечения этого active offer
- **AND** `ShopOffersPayload` содержит server tick, на котором был сформирован snapshot

#### Scenario: Лимитированный заказ не содержит данных таймера
- **WHEN** сервер формирует `ShopOffersPayload` для limited-заказа
- **THEN** соответствующий `ShopOfferEntry` помечен как не имеющий времени истечения

### Requirement: Временные заказы показывают таймер в списке
UI магазина SHALL показывать countdown до истечения для временных заказов в левой колонке списка заказов.

#### Scenario: Временный заказ виден в списке
- **WHEN** игрок открыл UI магазина и в левой колонке виден timed-заказ с будущим временем истечения
- **THEN** строка этого заказа показывает оставшееся время до истечения
- **AND** значение countdown уменьшается по мере прохождения игрового времени

#### Scenario: Лимитированный заказ виден в списке
- **WHEN** игрок открыл UI магазина и в левой колонке виден limited-заказ
- **THEN** строка этого заказа не показывает countdown-таймер

### Requirement: Выбранный временный заказ показывает синхронизированный таймер
UI магазина SHALL показывать countdown выбранного временного заказа в правой детальной панели, используя тот же расчёт остатка, что и левая колонка.

#### Scenario: Выбран временный заказ
- **WHEN** игрок выбирает timed-заказ в списке
- **THEN** правая детальная панель показывает оставшееся время до истечения этого заказа
- **AND** отображаемое значение соответствует countdown в строке этого же заказа в левой колонке

#### Scenario: Выбран лимитированный заказ
- **WHEN** игрок выбирает limited-заказ в списке
- **THEN** правая детальная панель не показывает countdown-таймер

### Requirement: Таймер временного заказа форматируется без лишнего сегмента часов
UI магазина SHALL форматировать countdown временного заказа как `M:SS`, когда до истечения остаётся меньше часа, и как `H:MM:SS`, когда остаётся час или больше.

#### Scenario: До истечения меньше часа
- **WHEN** игрок видит timed-заказ в списке или детальной панели
- **AND** до истечения заказа остаётся меньше 3600 секунд
- **THEN** countdown отображается в формате `M:SS`
- **AND** countdown не содержит сегмент часов

#### Scenario: До истечения час или больше
- **WHEN** игрок видит timed-заказ в списке или детальной панели
- **AND** до истечения заказа остаётся 3600 секунд или больше
- **THEN** countdown отображается в формате `H:MM:SS`

### Requirement: Истёкшие временные заказы нельзя обменять из устаревшего UI
Система MUST предотвращать успешный trade по timed-заказу, который истёк после открытия UI магазина.

#### Scenario: Временный заказ истекает при открытом UI
- **WHEN** timed-заказ истёк, пока UI магазина остаётся открытым
- **THEN** UI показывает этот заказ как истёкший или с нулевым оставшимся временем
- **AND** кнопка `Trade` неактивна для этого заказа

#### Scenario: Сервер получает trade для истёкшего временного заказа
- **WHEN** сервер получает `ShopTradeActionPayload` для заказа, который больше не является активным
- **THEN** сервер возвращает ошибку trade
- **AND** сервер не удаляет требуемые предметы из инвентаря игрока
- **AND** сервер не начисляет очки команде игрока

### Requirement: Иконки требуемых предметов показывают vanilla tooltip
UI магазина SHALL показывать стандартный vanilla tooltip предмета при hover по миниатюре требуемого предмета или блока.

#### Scenario: Наведение на иконку требования в списке
- **WHEN** игрок наводит курсор на миниатюру требуемого предмета в строке заказа левой колонки
- **THEN** UI показывает стандартный tooltip этого `ItemStack`, как в инвентаре Minecraft

#### Scenario: Наведение на иконку требования в детальной панели
- **WHEN** игрок наводит курсор на миниатюру требуемого предмета в правой детальной панели
- **THEN** UI показывает стандартный tooltip этого `ItemStack`, как в инвентаре Minecraft
